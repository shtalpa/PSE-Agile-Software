**Part A: Internal Alpha Testing**
To verify core functionality and link integrity before showing the prototype to outsiders. Each team member performed a click-through audit of all assigned modules.

Internal Findings:
1. Admin login failed to redirect on incorrect password entry. (Status: Fixed)
2. Level 4 button on Campus Map page was non-functional. (Status: Fixed)
3. Filter health percentages were inconsistent across the Station list. (Status: Fixed)
4. Admin dashboard navigation bar button was linked to the wrong pages. (Status: Fixed)
5. The filtering button on Admin's View Report page was non-functional. (Status: Fixed)

**Part B: User Testing (UAT)**
To gather feedback on UI/UX and accessibility from the student perspective.

Feedback and Iteration:
1. Home page navigation buttons were too small for responsive use.
   Corrective Action: Increased button dimensions.
2. Home page background color was overly bright and disrupt text visibility.
   Corrective Action: Standardized the background color palette to a lighter hue.
3. Users lacked a central location to view real-time system updates.
   Corrective Action: Added a notification bell icon to Home.
4. The "Offline" status icon failed to pulse in the Campus Map page.
   Corrective Action: Synchronized the animation logic.
5. Quick access toolbar in the Station page was pushed off-screen when cards expanded.
   Corrective Action: Redesigned the card expansion logic.
6. The "Active" filter button color was too dark in the Station page.
   Corrective Action: Adjusted the background to a lighter shade.
7. Typographical error identified in the Maintenance Report form.
   Corrective Action: Fixed the typing error.
